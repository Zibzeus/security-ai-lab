# Repository audit — 2026-06-23

## Scope

Reviewed all source, tests, scripts, deployment files, skills, policies, and
documentation in the supplied repository archive.

## Findings fixed

### High

1. **Approval self-assertion**
   - Any caller with the normal agent API key could submit
     `approved_capabilities`.
   - Fixed by requiring a separate `X-Approval-Key`.

2. **Denied engagement categories were not enforced**
   - `denied_categories` and `approval_categories` were loaded from YAML but not
     represented in the runtime object.
   - Fixed with deterministic category actions. Unknown categories now deny.
     `approved=true` cannot bypass denied categories.

3. **Generic shell would expose executor secrets if run directly**
   - Implemented only through bubblewrap with a minimal read-only filesystem and
     per-run writable artifact directory.
   - No unsandboxed fallback exists.

4. **Weak placeholder secrets were valid runtime defaults**
   - Agent and executor now fail startup unless required shared secrets are
     non-placeholder values of at least 32 characters.

### Medium

1. **Python packages could not be installed**
   - Setuptools discovered non-package top-level directories.
   - Fixed by explicitly declaring `app` and `executor` packages.

2. **CALDERA launch payload used an outdated flat shape**
   - Fixed to use nested `adversary`, `planner`, and `source` objects.

3. **CALDERA report retrieval was missing**
   - Added report and event-log retrieval using the official API v2 routes.

4. **BloodHound adapter was missing**
   - Added signed read-only Cypher query support.
   - Mutation keywords and multi-statement queries are rejected locally.

5. **SQLite connections were not explicitly closed**
   - Fixed connection lifecycle to prevent locked temporary databases,
     especially on Windows.

6. **BAS service bound to every interface**
   - systemd now uses configurable `EXECUTOR_BIND_HOST` and
     `EXECUTOR_PORT`.

7. **Artifact files could overwrite each other**
   - Each execution now gets a unique private run directory.

8. **Evidence files used default permissions**
   - Case directories use mode `0700`; audit, command, stdout, stderr, and API
     artifacts use `0600`.

9. **12 GB VM defaults were aggressive**
   - Initial llama.cpp context reduced to 4096 and batch to 128.

## Residual risks

- Generic shell target parsing cannot prove the runtime destination of encoded,
  downloaded, proxied, or dynamically resolved commands. BAS VM egress policy
  must enforce engagement networks.
- HMAC replay nonces are process-local. The 60-second timestamp window limits
  exposure, but a shared durable nonce store is needed for horizontally scaled
  executors.
- MCP tool names and product API behavior remain environment/version-specific.
- End-to-end CALDERA, BloodHound, ExtraHop, and CrowdStrike tests require the
  actual lab services and credentials.
- The llama.cpp container image should be pinned to a tested tag or digest
  after deployment validation.
